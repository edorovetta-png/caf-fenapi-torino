## Project Overview

Website and booking system for **Circolo FENAPI — Sede Provinciale di Torino** (tax assistance center). Two separate applications live in this repo:

- **`frontend/`** — Static HTML/CSS/JS showcase site presenting CAF services (730, ISEE, IMU, pensions, etc.)
- **`caffenapi/`** — React app (built with Lovable) for online appointment booking, admin dashboard, and analytics. Uses Supabase as backend.

Read `PROJECT_STATUS.md` for detailed component-by-component status. After significant changes, update it.

## Architecture

The repo follows a 3-tier agent architecture (documented in `Gemini.md`):

1. **Directives** (`directives/`) — SOPs in Markdown defining what to do
2. **Orchestration** — AI agent decision-making layer
3. **Execution** (`execution/`) — Deterministic Python scripts

Key principle: push complexity into deterministic code (`execution/`), not into LLM reasoning. Check `execution/` for existing scripts before writing new ones. Update directives when you learn something new (API limits, edge cases).

## Commands

### Caffenapi (React booking app)
```bash
cd caffenapi
npm install          # install dependencies
npm run dev          # dev server on port 8080
npm run build        # production build
npm run test         # run tests (vitest)
npm run test:watch   # run tests in watch mode
npm run lint         # eslint
```

### Execution scripts
```bash
pip install -r requirements.txt   # root requirements.txt
python execution/<script_name>.py
```

## Key Technical Details

### Static site (`frontend/`)
- Pure HTML/CSS/JS, no build step. Open `index.html` directly or serve with any static server.
- Design system lives in `styles.css` with CSS custom properties derived from `fenapi-group-brand-guidelines.md`.
- Service pages in `frontend/servizi/` were generated by `execution/generate_service_pages.py`.
- Google Analytics 4 (G-YNHWG87MSQ) is embedded in all pages.

### Booking app (`caffenapi/`)
- Vite + React 18 + TypeScript + shadcn/ui + Tailwind CSS
- Path alias: `@/` maps to `src/`
- Supabase handles auth, database (PostgreSQL), and Edge Functions
- 8 Edge Functions in `supabase/functions/`: appointment CRUD, availability checks, email queue, Google Calendar/Drive integration
- 12 database migrations in `supabase/migrations/`
- **Not tracked in the main git repo** — managed separately via Lovable

### Brand guidelines
Always consult `fenapi-group-brand-guidelines.md` before making visual changes. The primary color is `#206088`, font is Open Sans.

### Environment
- `.env` in root holds API keys (currently placeholders)
- `.tmp/` is for intermediate files — never commit, always regenerable
- `credentials.json` and `token.json` (Google OAuth) are gitignored

### Blog SEO (`frontend/blog/`)
- Blog articles use the same design system as service pages (styles.css + service-page.css)
- Each article has: H1, publication date, Schema.org Article structured data, Open Graph tags, CTA to prenotazioni.fenapipiemonte.org
- Blog index (`frontend/blog/index.html`) lists all articles with title, date, and preview
